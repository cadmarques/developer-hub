Order & Payment Processing System

Simulação de ecossistema de microserviços com Spring Boot 3, Java 17 e boas práticas de backend.

Visão Geral

Este projeto demonstra um sistema de processamento de pedidos e pagamentos, simulando a interação entre dois bounded contexts: Order e Payment.

Apesar de estar num único JAR, a arquitetura reflete como estes domínios seriam separados num ecossistema real de microserviços.

Tecnologias

  • Java 17
  • Spring Boot 3
  • Spring Web, Data JPA, Validation
  • H2 Database (in-memory)
  • JUnit 5, Mockito
  • Swagger / OpenAPI
  • Lombok (boas práticas)

Funcionalidades

  • Criação de pedidos com validação rigorosa
  • Processamento de pagamento simulado (70% sucesso, 30% falha)
  • Tratamento global de exceções com @ControllerAdvice
  • Endpoints REST bem estruturados
  • Testes automatizados (unitários e de integração)

Documentação da API (Swagger UI)

A API inclui documentação interativa com OpenAPI/Swagger, permitindo testar endpoints diretamente do browser.
